# ru-voice-command-recognition

Система распознавания русскоязычных голосовых команд для управления роботом — от микрофонного ввода до навигации по сетке с построением маршрута (A*).

В отличие от классического pipeline "ASR → NLU", распознавание команд строится напрямую на **аудиоэмбеддингах энкодера Whisper** (`openai/whisper-tiny`): из акустического представления извлекается компактный вектор признаков, поверх которого обучается лёгкий классификатор. Это позволяет обойтись без полной транскрипции речи и получить устойчивую классификацию команд при низкой задержке на CPU.

## Возможности

- **Распознавание команд в реальном времени** с микрофона (`realtime.py`): voice activity detection по порогу энергии, классификация и преобразование в действия робота.
- **Intent / slot-filling движок** (`core/intent_engine.py`): составные команды вида "ПОВЕРНИ → НАЛЕВО → ДВА" собираются по таймауту в единое намерение со слотами (направление, угол, число шагов), а не обрабатываются как изолированные слова.
- **Обучение классификатора** (`train.py`): разбиение по дикторам (train/test/TTS), кэширование эмбеддингов с проверкой версии и метаданных, аугментация аудио (изменение громкости, временной сдвиг, добавление шума, time-stretch).
- **Оценка качества**:
  - `evaluate.py` — точность и confusion matrix на новых дикторах;
  - `speaker_eval.py` — обобщение модели на голоса, не встречавшиеся при обучении;
  - `noise_eval.py` — устойчивость к шуму при разных уровнях SNR (от +30 до −5 дБ).
- **Экспорт и оптимизация инференса** (`export.py`): энкодер Whisper экспортируется в ONNX и запускается через ONNX Runtime; на CPU дополнительно применяется динамическая INT8-квантизация (PyTorch) для ускорения извлечения эмбеддингов.

## Архитектура

```
Микрофон → VAD (порог энергии) → Whisper-encoder (эмбеддинг 384d, ONNX/INT8)
         → классификатор (Keras) → Intent Engine (slot-filling) → действия робота
```

Поддерживается 24 команды: движение (вперёд/назад/влево/вправо/двигайся/вернись),
повороты (поверни/налево/направо/развернись/разворот), управление режимом
(автоматический/ручной/режим), числительные-слоты (один/два/три), включение/выключение,
изменение скорости (быстрее/медленнее) и др. — заданы в `config.py` (`COMMAND_ACTIONS`, `INTENTS`).

## Технологии

Python · PyTorch · TensorFlow/Keras · HuggingFace Transformers (Whisper) ·
ONNX Runtime · librosa · audiomentations · scikit-learn · sounddevice

## Данные

Собственный датасет голосовых команд: ~25 записей на класс (24 класса) от нескольких
дикторов + синтетические TTS-примеры, с разделением по дикторам на train/test, чтобы
оценка отражала обобщение на новые голоса, а не переобучение под конкретного диктора.

## Структура проекта

```
src/
├── core/
│   ├── preprocess.py     # извлечение эмбеддингов Whisper, аугментации, ONNX/INT8 инференс
│   ├── intent_engine.py  # slot-filling движок составных команд
│   ├── autopilot.py      # автономная навигация по маршруту (A*)
│   ├── grid_map.py       # сетка, препятствия, поиск пути
│   └── robot_viz.py      # визуализация
├── train.py              # обучение классификатора поверх эмбеддингов
├── evaluate.py           # оценка на новых дикторах + confusion matrix
├── speaker_eval.py       # генерализация на незнакомые голоса
├── noise_eval.py         # устойчивость к шуму (SNR sweep)
├── export.py             # экспорт Whisper-энкодера в ONNX
├── realtime.py           # инференс в реальном времени с микрофона
└── record.py             # запись обучающих сэмплов
```

## Запуск

```bash
pip install -r requirements.txt
python src/record.py        # запись датасета
python src/train.py         # обучение классификатора
python src/evaluate.py      # оценка качества
python src/realtime.py      # запуск распознавания в реальном времени
```
